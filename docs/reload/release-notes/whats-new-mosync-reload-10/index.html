What's New in MoSync Reload 1.0

MoSync Reload 1.0 (compatible with the MoSync SDK 3.2.1) provides enhanced functionality for rapid development of HTML5/JavaScript mobile apps across multiple mobile devices and simulators. This release also features a completely revamped Development UI to simplify and streamline the development workflow.

Use a WebKit-compatible browser: The Development UI in MoSync Reload 1.0 is designed for use with WebKit-compatible browsers such as Google Chrome and Apple Safari. In other browsers, such a Mozilla Firefox, the weinre tool is not available, and some functionality may not be available.

Update your existing projects and installed clients: I you have projects created with previous versions of MoSync Reload or MoSync SDK, make sure to update the wormhole.js file to the latest version. Also make sure to install the latest Reload Client on your devices/simulators. Please note that Reload Client 1.0 are NOT compatible with Reload 1.0.1. For Reload 1.0.1 you need to install the new Reload Client 1.0.1 app on your devices.

What is in this document?

This document provides an overview of what is new in Reload release 1.0 and 1.0.1. New functionality, important upgrade information, bug fixes, known limitations.

MoSync Reload consists of two major components, the Reload Client running on the device, and the Development UI running on a desktop machine. (The Development UI runs in a WebKit browser, and talks to the Reload Server that communicates with the Reload Client).

Revamped Development UI

The UI for developing Reload apps has been redesigned and updated. New tools such as the JavaScript Workbench and weinre has been added. There is also a new Quick Start guide to help you get started easily.

Updated Reload Client application

The Reload Client application has been updated with support for the new tools in the Development UI. The MoSync Reload 1.0 downloads includes native Reload Client projects for Android, iOS, and Windows Phone. The new clients are also available from online app stores and marketplaces. Please ensure that you have the latest Reload Client installed before starting to use Reload. The new Development UI and server work only with the new version of the client app.

Updating Existing Projects

For detailed instructions, please read the Wormhole Upgrade Guide.

Updating wormhole.js

For your existing projects to work in the new Reload version (and in MoSync SDK 3.2.1), you must update the file wormhole.js to the most recent version. Just make a copy of this file from a project created with the new version of Reload, and paste it into your existing project to replace the old version.

Updating MoSync SDK

If you want to build a native app from your Reload project that you can put on the App stores, you need to use MoSync SDK 3.2.1. Get it from the download page.

Updating your C++ code

If you have existing projects you build using MoSync SDK, you need to update them if they were created with MoSync SDK 3.1.1 or earlier versions. Class WebAppMoblet (deprecated) has been replaced by class HybridMoblet. If you just created an HTML5 project created from a template without modifying any C++ code, you can create a new project using Reload 1.0 or MoSync SKD 3.2.1 and paste your HTML/JS files and resources into the new project. That should work out-of-the-box.

New Wormhole Functionality

The MoSync JavaScript library has been updated with new functionality. Updating Wormhole apps is essential if you have existing apps developed with previous Reload or MoSync SDK versions.

Screen Orientation Support in JavaScript

There is a new function for setting the screen orientation of the device called mosync.app.screenSetOrientation(). This function allows you to set the screen orientation to portrait, landscape, or dynamic mode. Examples:

mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_DYNAMIC);
mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_PORTRAIT);
mosync.app.setScreenOrientation(mosync.SCREEN_ORIENTATION_LANDSCAPE);

Open External Web Browser from JavaScript

If you want to open an external website from your app, you can open a URL in the web browser of the device using the new function mosync.app.openExternalURL(). Opening the URL directly in a WebView widget is also possible, but may not provide the support users need, like web navigation. You can also use this function to dial a telephone number directly from your app. Examples:

mosync.app.openExternalURL("http://maps.google.com/");
mosync.app.openExternalURL("tel:046850510300");

Inspection Tool (weinre)

The inspection tool weinre (WEb INspector REmote) is an industry standard HTML/JavaScript tool which is useful for debugging HTML5/JavaScript apps. This tool is now available in the Reload Development UI. Using weinre, you can inspect and manipulate your domain object model (DOM), manage HTML5 databases and storage, and view JavaScript log messages with console.log. Check out the weinre online documentation.

Known limitations:

  • weinre supports WebKit browsers. This means the to use weinre, you must run the Development UI in a browser such as Chrome or Safari, and the Reload Client must run on Android or iOS (weinre is not currently not supported on Windows Phone).
  • When using weinre with Reload, currently only the main project file, index.html is inspectable.
  • Note that weinre is only enabled when the weinre tab is selected. If moving away from this tab (by selecting another tab), e.g. logging will not be functional until you return to it.
  • The client part of weinre takes some time to connect to the server, so initial console.log messages may not display in the weinre log console.

JavaScript Workbench

This tool lets you evaluate single or multiple JavaScript statements and immediately see the result. The Workbench gives you a very quick way to experiment with JavaScript code before adding it into your application. This is useful for both development and debugging. Any return values and error mesags are shown under the Log tab. See the JavavaScript Workbench Tutorial for further details.

Unified JavaScript Logging

Different devices output console.log messages in different ways. One way to get all of your console.log output display in the Reload Development UI (under the Log tab) is to define this at the start of your app (but after loading wormhole.js and other JavaScript libraries):

console.log = mosync.rlog;

Improved Power Usage by Hybrid Moblets

We have improved the power usage of the HybridMoblet in the Wormhole C++ Library, dramatically reducing battery drain when an app is sent to the background. To take advantage of this improvement, you will need to rebuild your apps using the new MoSync SDK 3.2.1. Note that this affects only applications that are build and deployed using the SDK. The new version of the Reload Client also have this fix enabled.

Quick Start Guide

The Reload Development UI has a new Quick Start guide to get you up and running in a matter of minutes. The guide shows you how to connect devices, create projects, edit source files, quickly reload apps, and work with the different tools in Reload.

Updated User Guides and Tutorials

All guides and tutorials for MoSync Reload have been updated for this new release and are available online in our Getting Started with MoSync Reload page.

Bugs and Known Limitations

Moving or deleting the current workspace folder. If the Reload workspace folder is moved or deleted, the Reload server will crash wen started. A fix for this is to delete (or edit) the file lastWorkspace.dat, which is find the the Reload server directory. This file contains a string with the last used workspace. (A Reload workspace is a directory that contains MoSync HTML5 projects.)

Duplicated main.cpp files. (This bug has been fixed in Reload 1.0.1.) There is a bug that causes two copies of the file main.cpp to be created you create a new project in Reload. As long as you use Reload, this causes no problems, but if you build the project in the MoSync SDK (using the MoSync Eclipse IDE) you will get a build error. The workaround is to delete the main.cpp file in the LocalFiles foler (and keep the one in the root folder of the project).
